## 通用 Response Envelope

所有端点返回的 JSON 都遵循同一 envelope:

```typescript
interface ResponseEnvelope<T> {
  success: boolean;        // true = 操作成功;false = 操作失败
  reason:  string | null;  // 失败原因码;成功时为 null
  elapsed_ms: number;      // 处理耗时(毫秒)
  data: T;                 // 端点专有 payload
  stderr?: string;         // 仅写端点遇 git 错误时存在(≤4 KB)
}
```

**HTTP status 规则**:
- `200` — 操作完成(无论 `success` 是 true / false,都返回 200)
- `304` — 仅 GET 端点 + ETag 缓存命中(无 body)
- `5xx` — 仅在服务端异常时出现;前端应重试

**认证**:所有端点仅 L1 管理员用户可用(普通用户即使知道 URL 也会收到 403)。Dashboard 部署时一般已 session 通过其他途径校验。

## 1. `POST /spcode/git-init`

**用途**:在指定目录上执行 `git init`(创建新 repo 或 bare repo)。**这是 webapi 中唯一豁免 `_git_endpoint_preflight` 的写端点**(因为"未加载项目" = "你要新建")。

**Request Body**:
```typescript
interface GitInitRequest {
  path: string;                       // 必填,绝对路径
  initial_branch?: string;            // 可选,默认 "main"
  bare?: boolean;                     // 可选,默认 false;true 时创建 bare repo
  force?: boolean;                    // 可选,默认 false;true 时允许在非空目录 init(v2.17.1 新增)
}
```

**成功响应(200)**:
```typescript
interface GitInitResponse extends ResponseEnvelope<{
  initialized:    true;
  path:           string;             // 实际创建的 repo 路径
  initial_branch: string;             // 实际设置的默认分支
  bare:           boolean;            // 是否 bare repo
  force:          boolean;            // 实际使用的 force(v2.17.1 新增)
  git_dir:        string;             // .git 目录路径
}> {}
```

**错误 reason**:
| reason | 含义 | 前端 UX 建议 |
|--------|------|--------------|
| `path_not_directory` | path 不存在或不是目录 | 高亮 path 输入框,提示必须存在且为目录 |
| `directory_not_empty` | 目录非空(且 `force=false`) | **v2.17.1:弹二次确认对话框,用户确认后用 `force=true` 重试** |
| `already_a_git_repo` | `.git/` 已存在(**hard-ban,`force=true` 不绕过**) | 提示该目录已是 git repo,引导用户加载项目而非 init |
| `path_unsafe` | path 含 `..` 或命中黑名单 | 提示路径非法 |
| `worktree_blacklisted` | path 命中 `file_remove_blacklist` 配置 | 提示该目录受保护 |
| `init_failed` | git 子进程失败 | 显示 stderr,提示用户检查 git 安装 |

**示例**:
```bash
POST /spcode/git-init
Content-Type: application/json

{"path": "D:/Users/dev/projects/myapp", "initial_branch": "main"}
```

```json
{
  "success": true, "reason": null, "elapsed_ms": 234,
  "data": {
    "initialized": true,
    "path": "D:/Users/dev/projects/myapp",
    "initial_branch": "main",
    "bare": false,
    "force": false,
    "git_dir": "D:/Users/dev/projects/myapp/.git"
  }
}
```

**`force=true` 二次确认示例(v2.17.1)**:
```bash
POST /spcode/git-init
Content-Type: application/json

{"path": "D:/Users/dev/projects/myapp", "force": true}
```

```json
{
  "success": true, "reason": null, "elapsed_ms": 198,
  "data": {
    "initialized": true,
    "path": "D:/Users/dev/projects/myapp",
    "initial_branch": "main",
    "bare": false,
    "force": true,
    "git_dir": "D:/Users/dev/projects/myapp/.git"
  }
}
```

> **前端 UX 模式(v2.17.1)**:收到 `directory_not_empty` 时,弹二次确认对话框
> ("目录非空,继续初始化不会删除现有文件,但它们将处于 untracked 状态。是否继续?"),
> 用户确认后用 `force=true` 重试。`already_a_git_repo` 是不可恢复错误,
> 引导用户去 `/spcode/project-status` 加载现有 repo。

## 2. `GET /spcode/git-branches`

**用途**:列出 local + remote 分支,标记 current + default。**支持 ETag/304 客户端缓存**(默认 max-age 1 秒)。

**Query 参数**:
- `umo?: string` — 用户会话 ID(必填,有 umo 后端才会查找已加载项目)
- `worktree?: string` — worktree 路径(可选,默认 primary)

**Request Headers**(可选):
- `If-None-Match: "<etag>"` — 客户端缓存的 ETag;匹配则后端返回 304 + 无 body

**成功响应(200)**:
```typescript
interface BranchInfo {
  name:               string;   // e.g. "main" / "feature/x" / "origin/HEAD"
  is_local:           boolean;
  is_remote:          boolean;
  current:            boolean;  // 本仓库当前所在分支(可能同时是 true)
  upstream?:          string;   // e.g. "origin/main"
  upstream_tracking?: "ahead" | "behind" | "diverged" | "synced";
}

interface GitBranchesResponse extends ResponseEnvelope<{
  branches: BranchInfo[];
  default:  string;             // git config init.defaultBranch,如 "main"
  detached: boolean;            // detached HEAD 状态
  total:    number;             // branches.length
}> {}
```

**响应 Headers**(GET 端点):
- `ETag: "<hash>"` — 弱 ETag(基于 current + default + 列表内容 hash)
- `Cache-Control: private, max-age=1`

**错误 reason**(通用 preflight):
- `no_project_loaded` — 提示用户先 git-init 或加载
- `not_a_git_repo` — 提示该目录不是 git repo
- `worktree_invalid` — `?worktree=` 路径非法或跨仓库

**示例**:
```bash
GET /spcode/git-branches?umo=session%3A123
If-None-Match: "a1b2c3d4"
```

## 3. `POST /spcode/git-branch-create`

**用途**:从当前 HEAD(默认)或指定 `start_point` 创建新分支(`git branch <name>`)。

**Request Body**:
```typescript
interface GitBranchCreateRequest {
  name:         string;   // 必填,符合 git ref-name 规范(字母数字/_/-/.等,不以 - 开头)
  start_point?:  string;   // 可选,默认 HEAD;支持分支名 / SHA / 任意 git rev
  force?:       boolean;  // 可选,默认 false;若分支已存在则报错(不再静默)
}
```

**成功响应(200)**:
```typescript
interface GitBranchCreateResponse extends ResponseEnvelope<{
  created:     true;
  name:        string;     // 新分支名
  start_point: string;     // 实际使用的 start_point(回显)
  forced:      boolean;    // 实际 force
  sha:         string;     // 新分支顶端 SHA(40 hex 字符)
}> {}
```

**错误 reason**:
| reason | 含义 | UX |
|--------|------|-----|
| `invalid_branch` | name 不匹配 git ref-name 规范 | 高亮 name 输入框,提示合法字符 |
| `branch_exists` | 分支已存在(`force=false`) | 提示用 `force=true` 覆盖 |
| `ref_not_found` | `start_point` 不存在 | 提示用户刷新分支/历史,选择正确 ref |
| `worktree_dirty` | worktree 有未提交改动 | 提示先 commit/stash |

**示例**:
```bash
POST /spcode/git-branch-create

{"name": "feature/payment", "start_point": "main", "force": false}
```

## 4. `POST /spcode/git-branch-delete`

**用途**:删除分支(`-d` 仅 merged,`-D` 含 unmerged)。**`branch_is_current` 是硬禁,`force=true` 不绕过**。

**Request Body**:
```typescript
interface GitBranchDeleteRequest {
  name:  string;          // 必填
  force?: boolean;        // 默认 false → 仅删已合并;true → -D 强制
}
```

**成功响应(200)**:
```typescript
interface GitBranchDeleteResponse extends ResponseEnvelope<{
  deleted: true;
  name:    string;
  force:   boolean;
  merged:  boolean;        // 分支是否已 merged 到 current
  sha:     string;          // 被删分支尖端 SHA
}> {}
```

**错误 reason**:
| reason | 含义 | UX |
|--------|------|-----|
| **`branch_is_current`** | **硬禁 — `force=true` 不绕过** | 提示用户先 switch 到别的分支 |
| `branch_not_merged` | `-d` 模式但分支未 merged,且 `force=false` | 提示用户用 `force=true` 强制删 |
| `branch_not_found` | 分支不存在 | 提示可能已被删,刷新列表 |
| `worktree_dirty` | worktree 有未提交改动(`force=false`) | 提示先提交 |
| `worktree_missing_local` | 当前 worktree 缺少本地分支 | 罕见,提示用户报 bug |
| `branch_checked_out_other_worktree` | 分支被其他 worktree 检出 | 提示用户在对应 worktree 删除 |

**示例**:
```bash
POST /spcode/git-branch-delete
{"name": "feature/old", "force": false}
```

## 5. `POST /spcode/git-branch-switch`

**用途**:切换当前分支或创建+切换(`git switch <name>`)。支持 `create` / `detach` / `force` 跨字段组合。

**Request Body**:
```typescript
interface GitBranchSwitchRequest {
  name:         string;        // 必填
  create?:      boolean;       // 默认 false
  start_point?: string;        // 仅 `create=true` 时合法
  force?:       boolean;       // 默认 false;create=true 时此字段被忽略
  detach?:      boolean;       // 默认 false
}
```

**跨字段约束**(后端自动校验):
- `create=true AND detach=true` → **互斥** → 报 `invalid_body`
- `create=true AND force=true` → 被忽略(`force` 仅切换时生效)

**成功响应(200)**:
```typescript
interface GitBranchSwitchResponse extends ResponseEnvelope<{
  switched: true;
  name:     string;         // 切换后的分支名
  previous: string;         // 切换前的分支名(可能是 "DETACHED")
  created:  boolean;        // 本次请求是否同时创建了新分支
  force:    boolean;        // 实际使用的 force
  detach:   boolean;
}> {}
```

**错误 reason**:
| reason | 含义 | UX |
|--------|------|-----|
| `invalid_branch` | name 非法 | 同上 |
| `branch_not_found` | 切换目标不存在(且未 create) | 提示用户先 create 或选择存在分支 |
| `branch_exists` | `create=true` 但分支已存在 | 提示取消 create 或选别的 name |
| `worktree_dirty` | worktree 有未提交改动(`force=false`) | 提示先 commit / stash |
| `invalid_body` | 跨字段组合冲突(create+detach) | 前端应在自己 UI 上阻止这种组合 |

**示例**:
```bash
POST /spcode/git-branch-switch
{"name": "feature/x"}
```

## 6. `POST /spcode/git-revert`

**用途**:生成回滚 commit(`git revert <ref> --no-edit`)。复用 `git_commit` 的错误分类与 env 透传。

**Request Body**:
```typescript
interface GitRevertRequest {
  ref?:      string;       // 默认 "HEAD",必须是 commit
  no_edit?:  boolean;      // 强制 true(headless 服务不开编辑器)
}
```

**前置条件**(后端强制):
- `ref` 必须能解析为 commit(`git rev-parse <ref>^{commit}` 通过)
- worktree 必须 clean(`git status --porcelain` 为空)
- `no_edit` 必须为 true

**成功响应(200)**:
```typescript
interface GitRevertResponse extends ResponseEnvelope<{
  reverted:        true;
  ref:             string;          // 传入的 ref
  revert_sha:      string;          // 新生成的 revert commit SHA(40 hex)
  revert_message:  string;          // 默认 "Revert \"<original-message>\""
  files_touched:   string[];        // revert commit 修改/删除的文件列表
  directory:       string;
  umo:             string;
  worktree:        string;
}> {}
```

**错误 reason**:
| reason | 含义 | UX |
|--------|------|-----|
| `commit_not_found` | `ref` 不解析为 commit | 提示用户刷新 git-log,选择有效 commit |
| `worktree_dirty` | worktree 有未提交改动 | 提示先 commit/stash |
| `empty_repository` | 仓库无任何 commit | 提示用户先做 commit |
| **`hook_rejected`** | pre-commit hook 失败 | 显示 stderr,提示用户看 hook 输出 |
| **`identity_not_set`** | git `user.email` / `user.name` 未设 | 显示 stderr,引导运行 `git config user.email "..."` |
| **`revert_conflict`** | revert 与当前改动冲突 | 引导用户到 IDE 解决冲突 |
| `nothing_to_revert` | 该 commit 无可反向应用修改 | 提示所选 commit 已经是空操作(如 merge commit) |
| `git_error` | 其他 git 错误 | 显示 stderr |
| `invalid_param` | `no_edit=false`(强制 true) | 前端应只发 `no_edit=true` |

**环境变量透传**:后端会自动透传 `GIT_AUTHOR_*` / `GIT_COMMITTER_*`(用于测试或集成方注入测试身份,**前端无需感知**)。

**示例**:
```bash
POST /spcode/git-revert
{"ref": "a1b2c3d4e5f6...", "no_edit": true}
```

```json
{
  "success": true, "reason": null, "elapsed_ms": 312,
  "data": {
    "reverted": true,
    "ref": "a1b2c3d4e5f6...",
    "revert_sha": "e5f6g7h8i9j0...",
    "revert_message": "Revert \"feat: add payment gateway\"",
    "files_touched": ["src/payment.py"],
    "directory": "D:/.../repo",
    "umo": "session:123",
    "worktree": "D:/.../repo"
  }
}
```

## TypeScript / fetch 集成代码示例

```typescript
// ── 类型定义(从 spec 抽取)─────────────────────────────────────
type ReasonCode = string;  // 见上表

interface ResponseEnvelope<T> {
  success:    boolean;
  reason:     ReasonCode | null;
  elapsed_ms: number;
  data:       T;
  stderr?:    string;
}

// ── 通用 fetcher(自带 304 处理 + 自动 throw)──────────────────
class SpcodeClient {
  async call<TReq, TRes extends ResponseEnvelope<unknown>>(
    path: string,
    method: 'GET' | 'POST' | 'PATCH' | 'DELETE',
    body?: TReq,
    headers: Record<string, string> = {},
  ): Promise<TRes & { headers: Headers } | { status: 304; headers: Headers }> {
    const finalHeaders: Record<string, string> = {
      ...(body ? { 'Content-Type': 'application/json' } : {}),
      ...headers,
    };
    const resp = await fetch(path, {
      method,
      headers: finalHeaders,
      body: body ? JSON.stringify(body) : undefined,
    });
    if (resp.status === 304) {
      return { status: 304 as const, headers: resp.headers };
    }
    const data = (await resp.json()) as TRes;
    return { ...data, headers: resp.headers };
  }

  // 调用示例:创建分支,失败自动按 reason 抛异常 / 调用 UX handler
  async gitBranchCreate(body: {
    name: string; start_point?: string; force?: boolean;
  }) {
    const r = await this.call('/spcode/git-branch-create', 'POST', body);
    if ('status' in r && r.status === 304) throw new Error('Unexpected 304');
    if (!r.success) {
      // 前端 UX dispatcher 示例
      switch (r.reason) {
        case 'branch_exists':
          throw new UxError('该分支已存在', { force: '请勾选 force 选项重试' });
        case 'invalid_branch':
          throw new UxError('分支名非法', { pattern: '^[A-Za-z0-9._/-]+$' });
        case 'ref_not_found':
          throw new UxError('start_point 不存在', { refresh: 'git-log' });
        default:
          throw new Error(r.reason ?? 'unknown');
      }
    }
    return r.data;
  }
}

// ── GET 端点带 ETag 缓存示例 ────────────────────────────────
async function listBranchesWithCache(umo: string, client: SpcodeClient) {
  const etag = localStorage.getItem(`etag:git-branches:${umo}`) ?? undefined;
  const r = await client.call<unknown, ResponseEnvelope<GitBranchesData>>(
    `/spcode/git-branches?umo=${encodeURIComponent(umo)}`,
    'GET',
    undefined,
    etag ? { 'If-None-Match': etag } : {},
  );
  if ('status' in r && r.status === 304) {
    // 缓存命中:复用 localStorage 的上一次响应数据
    return JSON.parse(localStorage.getItem(`data:git-branches:${umo}`) ?? '{}');
  }
  const newEtag = r.headers.get('ETag');
  if (newEtag) localStorage.setItem(`etag:git-branches:${umo}`, newEtag);
  localStorage.setItem(`data:git-branches:${umo}`, JSON.stringify(r.data));
  return r.data;
}

// ── UX Error 类(便于前端统一处理)─────────────────────────────
class UxError extends Error {
  constructor(
    public readonly message: string,
    public readonly hint?: Record<string, string>,
  ) {
    super(message);
  }
}
```
