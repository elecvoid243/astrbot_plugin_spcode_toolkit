## 1. 变更摘要

`POST /spcode/git-init` 新增一个可选请求字段 `force: bool`(默认 `false`)。

| 变更项 | v2.17.0(变更前) | v2.17.1(变更后) |
|--------|-------------------|-------------------|
| 请求体 | `{path, initial_branch?, bare?}` | `{path, initial_branch?, bare?, force?}` |
| 非空目录行为 | 一律拒绝(`directory_not_empty`) | `force=true` 时允许 init |
| 成功响应字段 | `initialized, path, initial_branch, bare, git_dir, umo, worktree` | **新增** `force` 字段 |
| `already_a_git_repo` | 拒绝 | 拒绝(**`force` 不绕过**,hard-ban) |

---

## 2. 请求体变更

### 新增字段

```typescript
interface GitInitRequest {
  path: string;                       // 必填,绝对路径(不变)
  initial_branch?: string;            // 可选,默认 "main"(不变)
  bare?: boolean;                     // 可选,默认 false(不变)
  force?: boolean;                    // ⭐ 新增(v2.17.1),默认 false
}
```

**`force` 字段语义**:
- `force=false`(默认):非空目录 → 返回 `directory_not_empty` 错误(v2.17.0 行为不变)
- `force=true`:非空目录 → **跳过非空检查,直接 init**(现有文件变为 untracked)

**`force=true` 不绕过的检查**(这些在 `force=true` 时仍然生效):

| 目录状态 | `force=false` | `force=true` |
|----------|---------------|--------------|
| 空目录 | ✅ init | ✅ init |
| 非空目录(无 `.git/`) | ❌ `directory_not_empty` | ✅ **跳过检查,直接 init** |
| 已有 `.git/` | ❌ `already_a_git_repo` | ❌ `already_a_git_repo`(**hard-ban**) |
| 路径不存在 | ❌ `path_not_directory` | ❌ `path_not_directory` |
| 路径含 `..` / 绝对路径 / symlink | ❌ `path_unsafe` | ❌ `path_unsafe` |
| 路径命中黑名单 | ❌ `worktree_blacklisted` | ❌ `worktree_blacklisted` |

### 类型校验

`force` 必须是 **严格 `boolean`** 类型。以下值会被拒绝(返回 `invalid_param`):

| 输入值 | 结果 | reason |
|--------|------|--------|
| `true` / `false` | ✅ 接受 | - |
| 不传(缺省) | ✅ 等同 `false` | - |
| `"true"`(字符串) | ❌ 拒绝 | `invalid_param` |
| `1`(整数) | ❌ 拒绝 | `invalid_param` |
| `null` | ❌ 拒绝 | `invalid_param` |

> ⚠️ 前端发送请求时务必使用 JSON `true` / `false`,不要发送字符串。

## 3. 响应体变更

### 成功响应(200)— 新增 `force` 字段

```typescript
interface GitInitSuccessResponse {
  success: true;
  reason: null;
  elapsed_ms: number;
  data: {
    initialized:    true;
    path:           string;      // 实际 init 的目录路径(不变)
    initial_branch: string;      // 实际默认分支(不变)
    bare:           boolean;     // 是否 bare repo(不变)
    force:          boolean;     // ⭐ 新增(v2.17.1):实际使用的 force 值
    git_dir:        string;      // .git 目录路径(不变)
    umo:            string | null;
    worktree:       string;      // 恒为 ""
  }
}
```

**前端适配**:
- `data.force` 为新增字段,前端读取时使用可选链或 fallback:
  ```typescript
  const force = response.data.force ?? false;
  ```
- 如果前端不需要展示 `force`,可以完全忽略此字段(向后兼容)。

### 失败响应 — 无新增 reason

`force=true` 不产生新的错误码。`force` 类型校验失败时复用已有的 `invalid_param`。

## 4. 前端推荐 UX 模式

### 场景:用户在非空目录上点击 "Git Init"

```
用户输入路径: D:/projects/myapp (非空)
                ↓
前端发送: POST /spcode/git-init  { "path": "D:/projects/myapp" }
                ↓
后端返回: { "success": false, "reason": "directory_not_empty", ... }
                ↓
前端弹出二次确认对话框:
  ┌─────────────────────────────────────────────────┐
  │  ⚠️ 目录非空                                     │
  │                                                   │
  │  D:/projects/myapp 目录中已有文件。               │
  │  初始化 Git 仓库不会删除现有文件,                 │
  │  但它们将处于 untracked 状态。                    │
  │                                                   │
  │  是否继续?                                        │
  │                                                   │
  │       [取消]              [确认初始化]            │
  └─────────────────────────────────────────────────┘
                ↓ 用户点击"确认初始化"
前端发送: POST /spcode/git-init  { "path": "D:/projects/myapp", "force": true }
                ↓
后端返回: { "success": true, "data": { "initialized": true, "force": true, ... } }
```

### 伪代码参考

```typescript
async function gitInit(path: string, opts?: {
  initialBranch?: string;
  bare?: boolean;
  force?: boolean;
}): Promise<GitInitResponse> {
  const body: GitInitRequest = {
    path,
    initial_branch: opts?.initialBranch ?? "main",
    bare: opts?.bare ?? false,
  };

  const res = await fetch("/spcode/git-init", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(body),
  });
  const json = await res.json();

  // v2.17.1: 非空目录时引导用户二次确认
  if (!json.success && json.data?.reason === "directory_not_empty") {
    const confirmed = await showConfirmDialog({
      title: "目录非空",
      message: `"${path}" 目录中已有文件。初始化 Git 仓库不会删除现有文件,但它们将处于 untracked 状态。是否继续?`,
      confirmText: "确认初始化",
      cancelText: "取消",
    });

    if (!confirmed) {
      return json; // 用户取消,返回原始错误
    }

    // 用 force=true 重试
    return gitInit(path, { ...opts, force: true });
  }

  return json;
}
```

### 不可恢复错误(不需要二次确认)

收到以下 reason 时,`force=true` **无法**解决,不要提供重试选项:

| reason | 前端处理 |
|--------|----------|
| `already_a_git_repo` | 提示"该目录已是 Git 仓库",引导用户去加载项目(`GET /spcode/project-status`)而非 init |
| `path_not_directory` | 高亮路径输入框,提示"路径不存在或不是目录" |
| `path_unsafe` | 提示"路径非法(含 `..` 或受保护)" |
| `worktree_blacklisted` | 提示"该目录受保护,禁止操作" |

## 6. 兼容性矩阵

| 前端版本 | 不传 `force` | 传 `force=true` | 读取 `data.force` |
|----------|-------------|-----------------|-------------------|
| v2.17.0 前端(不感知 force) | ✅ 正常工作(后端默认 false) | N/A(前端不会发) | ⚠️ 字段不存在,需 `?? false` fallback |
| v2.17.1 前端(感知 force) | ✅ 正常工作 | ✅ 正常工作 | ✅ 直接读取 |

**结论**:本次变更是**纯增量**。v2.17.0 前端无需任何改动即可继续工作,
v2.17.1 前端可选择性接入 `force` 以提供更好的非空目录 UX。
